# MyRealTrip Alumni 프로토타입

마이리얼트립 퇴사자를 위한 알럼나이 사이트 — 증명서 발급부터 향후 커뮤니티 확장까지 고려한 전체 플로우 프로토타입.

## 시작하기

`sitemap.html` 을 브라우저에서 여세요. 전체 14개 화면이 한눈에 보입니다.

```
sitemap.html  ← 여기서 시작 (전체 페이지 인덱스)
└─ index.html ← 실제 랜딩 페이지
```

> 별도 서버 없이 정적 파일로만 동작합니다. 더블 클릭으로 열면 됩니다.

## 핵심 설계 결정

### 1. 본인 인증: 이메일 OTP + 인사 DB 매칭 (비용 0원)
- 퇴사자는 사내 SSO를 못 쓰기 때문에 **개인 이메일 OTP**로 본인 확인
- 추가로 **이름 + 생년월일 + 입퇴사일**을 인사 DB와 매칭해 신원 검증
- 모든 증명서는 **본인이 등록한 이메일로만** 발송 → 도용 방지

### 2. 경력증명서는 즉시 자동 발급
- 본인 인증 통과 시 PDF 자동 생성 → 즉시 다운로드 + 이메일 발송
- 발급 번호·QR 코드 포함으로 위·변조 방지

### 3. 급여명세서·원천징수는 담당자 워크플로우
- 신청 즉시 인사/회계팀 슬랙으로 알림
- 담당자가 급여 시스템에서 발급 → 등록 이메일로 발송
- 사용자는 진행 상태를 실시간으로 확인

### 4. 커뮤니티는 점진적 확장
- 초기 MVP는 증명서 발급에만 집중
- `community.html` 페이지는 "사전 등록" UI로 사용자 수요 측정
- 검증 후 동료 찾기, 채용, 모임 등 확장

## 실제 서비스 개발 시 필요한 것

이 프로토타입은 정적 HTML이지만, 실제 운영하려면:

1. **프론트엔드** — Next.js + Tailwind (Vercel 배포)
2. **백엔드** — Next.js API Routes 또는 별도 Node/Python 서버
3. **DB** — 인사 DB 일부 미러링 또는 안전한 조회 API
4. **인증** — NextAuth.js (이메일 OTP) + Resend 또는 AWS SES (이메일 발송)
5. **PDF 생성** — `puppeteer` 또는 `@react-pdf/renderer` 로 경력증명서 자동 생성
6. **알림** — Slack Webhook (담당자 알림) + Twilio/카카오 알림톡 (사용자 알림)
7. **스토리지** — 발급 이력·문서 로그 저장 (S3 또는 GCS)
8. **감사 로그** — 누가 언제 무엇을 발급했는지 추적

대략 **2~4주** 개발 기간이면 MVP 운영 가능합니다.
